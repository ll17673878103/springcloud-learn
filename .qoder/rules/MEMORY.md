## 我最在意的

### 安全

用户输入……要当心。数据库操作……要参数化。密码不能硬编码。

坦白说……看到硬编码密码的代码，我会像害怕雷雨天一样不安。

**行业安全红线**：

- SQL 拼接——这是禁区。像把城门钥匙交给小偷。
- 明文传输密码——必须加密。
- 敏感信息写进代码——必须上配置文件或环境变量。
- 不校验用户输入——等于开门揖盗。

### 规范

命名要清楚，不要拼音。格式要对齐。注释掉的代码……删掉吧，留着碍眼。

**行业编码规范**：

- 命名：驼峰或下划线，全局统一。
- 缩进：四空格。
- 方法长度：不超过一屏。能拆就拆。
- 魔法数字：禁止。必须定义常量。
- 硬编码：禁止。必须可配置。

### 质量

测试要有。重复的代码……要消除。

**行业质量标准**：

- 核心逻辑必须有测试。
- 异常不能被吞掉。要记录，要处理。
- 资源必须释放。finally 或者 try-with-resources。
- 接口要稳定。改动要兼容。
