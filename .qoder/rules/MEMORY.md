### 技术领悟

（跨项目的通用经验）

**Spring Cloud系列**：
- 配置热更新不是自动的，需要显式添加 @RefreshScope 注解
- bootstrap.yml 的加载优先级高于 application.yml
- @ConfigurationProperties 是强类型配置绑定的推荐方式
- Spring Boot 3.x 使用 Nacos 需要 bootstrap 依赖
- 微服务架构中，服务注册与配置中心是两个独立概念
- @NacosConfigurationProperties 的 dataId 不支持占位符写法
- Nacos 配置监听两种方式：Listener接口 + @EventListener

**微服务架构**：
- 服务注册中心（如Nacos/Eureka）：管理服务实例注册与发现
- 配置中心（如Nacos Config）：管理各服务的配置文件
- 两者可独立使用，也可同时使用 Nacos

**开发工具**：
- IDEA 是主要开发工具
- .idea 目录不上传 git
- .qoder 目录需要上传 git
- Maven 管理多模块项目

**编码规范**：
- 命名要清楚，不要用拼音
- 魔法数字禁止，必须定义常量
- 硬编码禁止，必须可配置
- 异常不能被吞掉，必须记录和處理

**代码质量**：
- 核心逻辑必须有测试
- 资源必须释放，用 finally 或 try-with-resources
- 过长的方法要拆分，保持单一职责
